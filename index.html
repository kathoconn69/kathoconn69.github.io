CSS Selectors and Specificity

CSS Simple Selectors

Selector Definition Syntax
Type A type selector selects all elements of the given type within a document. element {style properties}
Class A class selector matches elements based on their given class attribute. .class_name {style properties}
ID An ID selector matches elements based on the value of their given ID attribute. #id_value {style properties}
Universal Selector A universal selector selects all HTML elements on the page. * {style properties}
element, element,... Selects all <div> and <p> elements div, p
element.class Selects only <p> elements with class='intro' p.intro

CSS Combinators

To minimize the code on a page, combine CSS selectors using combinators. A combinator explains the relationship between two elements.

Combinator Definition Syntax
Descendant Selector The descendant selector matches all elements that are descendants of a specified element, e.g., all <p> elements inside <div> elements. (space)
Child Selector The child selector selects all elements that are the children of a specified element, e.g., all <p> elements that are children of a <div> element. >
Adjacent Sibling Selector The adjacent sibling selector is used to select an element that is immediately after another specific element, e.g., the first <p> elements immediately after <div> elements. +
General Sibling Selector The general sibling selector selects all elements that are next siblings of a specified element, e.g., all <p> elements that are next siblings of <div> elements. ~

Specifity

If there are two or more CSS rules that point to the same element, the selector with the highest specificity value will "win," and its style declaration will be applied to that HTML element.

There are four categories that define the specificity of a selector:

  • Inline styles - Example: <h1 style="color: pink;">
  • IDs - Example: #navbar
  • Classes, pseudo-classes, attribute selectors - Example: .test, :hover, [href]
  • Elements and pseudo-elements - Example: h1, :before

To calculate specifity, start at 0, add 100 for each ID value, add 10 for each class value (or pseudo-class or attribute selector), add 1 for each element selector or pseudo-element.

Selector Specifity Value Calculation
p 1 1
p.test 11 1 + 10
p#demo 101 1 + 100
<p style="color: pink;"> 1000 1000
#demo 100 100
.test 10 10
p.test1.test2 21 1 + 10 + 10
#navbar p#demo 201 100 + 1 + 100
* 0 0 (the universal selector is ignored)

More Specifity Rule Examples:

  • Equal specificity: the latest rule wins - If the same rule is written twice into the external style sheet, then the latest rule wins:
    div#a {background-color: green;}
    #a {background-color: yellow;}
    div[id=a] {background-color: blue;}

    The first rule is more specific than the other two, and will therefore be applied.

  • ID selectors have a higher specificity than attribute selectors.
  • Contextual selectors are more specific than a single element selector - The embedded style sheet is closer to the element to be styled.
  • A class selector beats any number of element selectors - a class selector such as .intro beats h1, p, div, etc.
  • The universal selector (*) and inherited values have a specificity of 0 - The universal selector (*) and inherited values are ignored!
